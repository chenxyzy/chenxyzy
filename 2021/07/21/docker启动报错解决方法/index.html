docker启动报错解决方法

学习docker的时候，执行docker ps出现：

root@ubuntu:~# docker ps
Cannot connect to the Docker daemon. Is ‘docker -d’ running on this host?

执行docker -d 报错：

root@ubuntu:~# sudo docker -d
ERRO[0000] [graphdriver] prior storage driver “aufs” failed: driver not supported
FATA[0000] Error starting daemon: error initializing graphdriver: driver not supported

看了下系统启动日志，docker启动报错，看来是docker的守护进程没起来：

Jan 22 21:30:16 ubuntu docker[649]: time=”2016-01-22T21:30:16.502135073-08:00” level=error msg=”[graphdriver] prior storage driver "aufs" failed: driver not supported”
Jan 22 21:30:16 ubuntu docker[649]: time=”2016-01-22T21:30:16.502363312-08:00” level=fatal msg=”Error starting daemon: error initializing graphdriver: driver not supported”
Jan 22 21:30:07 ubuntu systemd[1]: docker.service: Main process exited, code=exited, status=1/FAILURE
Jan 22 21:30:07 ubuntu systemd[1]: docker.service: Unit entered failed state.
Jan 22 21:30:07 ubuntu systemd[1]: docker.service: Failed with result ‘exit-code’.

关于docker进程没起来的原因可能会有很多,比如：模块没加载，系统内核版本过低，硬盘空间不足等等。百度搜了一圈，国内大多都是抄来抄去，大多雷同，谷歌了一番，英文不太好，找了到一篇解决博客，参考：

http://log.rowanto.com/broken-docker-in-debian-jessie/

解决方法：

sudo mv /var/lib/docker/aufs /var/lib/docker/aufs.old
sudo service docker start

或者是执行：

docker -d -s aufs –storage-driver=overlay > /dev/null 2>&1 &
