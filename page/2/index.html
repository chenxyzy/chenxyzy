Docker 常用命令

拉取一个叫debian的镜像

sudo docker pull debian
# 查看本地已经有哪些镜像
sudo docker images

# 启动镜像
sudo docker run debian echo "hello world!"

# 让我们再看看我们有哪些镜像了。
# 注意每条命令执行后容器都会停止
# 可以看到有一个debian容器
sudo docker ps -a

# 现在，可以提交修改了。
# 提交后会看到一个新的镜像debian/helloworld
#  <CONTAINER ID> 是刚刚修改容器后得到的ID
sudo docker commit <CONTAINER ID> debian/helloworld

# 再看看我们有哪些镜像。
# 我们现在同时有debian和debian/helloworld镜像了。
sudo docker images

# 我们执行以下命令，看看这两个镜像有什么不同
sudo docker run debian [ -d /tmp/test ] && echo 'Directory found' || echo 'Directory not found'
# 如果/tmp/下有test文件夹就执行(echo "Directory found")否则就执行(echo "Directory not found")
sudo docker run debian/helloworld [ -d /tmp/test ]&& echo 'Directory found' || echo 'Directory not found'
# 同上

下面是如何持久化这些修改的一些命令。

导出(Export)
Export命令用于持久化容器（不是镜像）。所以需要通过以下方法得到容器ID：

sudo docker ps -a

接着执行导出：

sudo docker export > /home/debian-export.txz

保存(Save)

Save命令用于持久化镜像（不是容器）。所以需要通过以下方法得到镜像名称：

sudo docker images

接着执行保存：

sudo docker save debian> /home/debian-save.txz

查看所有的容器

sudo docker ps -a



# 删除它们
sudo docker rm <CONTAINER ID> 
# 删除所有的容器
sudo docker rm $(docker ps -a -q) 



# 查看所有的镜像
sudo docker images
 

# 删除它们
sudo docker rmi debian
sudo docker rmi debianhelloworld
# 删除所有的镜像
sudo docker rmi $(docker images -q)

导入容器：

导入debian-export.txz文件

cat /home/debianexport.txz | sudo docker import - debian-export:latest



# 查看镜像
sudo docker images



# 检查是否导入成功，就是启动一个新容器，检查里面是否存在/home/test目录（是存在的）
sudo docker run debian-export [ -d /home/test ]&& echo 'Directory found' || echo 'Directory not found'

导入镜像：

导入debian-save.tar文件

docker load < /home/debian-save.txz



# 查看镜像
sudo docker images



# 检查是否导入成功，就是启动一个新容器，检查里面是否存在/home/test目录（是存在的）
sudo docker run debian-save [ -d /home/test ]&& echo 'Directory found' || echo 'Directory not found'

那，它们之间到底存在什么不同呢？
导出后，会丢失历史和元数据。执行下面的命令就知道了：

显示镜像的所有层(layer)

sudo docker images --tree

执行命令，显示下面的内容。正你看到的，导出后再导入(exported-imported)的镜像会丢失所有的历史，而保存后再加载（saveed-loaded）的镜像没有丢失历史和层(layer)。这意味着使用导出后再导入的方式，你将无法回滚到之前的层(layer)，同时，使用保存后再加载的方式持久化整个镜像，就可以做到层回滚（可以执行docker tag 来回滚之前的层）。

docker启动报错解决方法

学习docker的时候，执行docker ps出现：

root@ubuntu:~# docker ps
Cannot connect to the Docker daemon. Is ‘docker -d’ running on this host?

执行docker -d 报错：

root@ubuntu:~# sudo docker -d
ERRO[0000] [graphdriver] prior storage driver “aufs” failed: driver not supported
FATA[0000] Error starting daemon: error initializing graphdriver: driver not supported

看了下系统启动日志，docker启动报错，看来是docker的守护进程没起来：

Jan 22 21:30:16 ubuntu docker[649]: time=”2016-01-22T21:30:16.502135073-08:00” level=error msg=”[graphdriver] prior storage driver "aufs" failed: driver not supported”
Jan 22 21:30:16 ubuntu docker[649]: time=”2016-01-22T21:30:16.502363312-08:00” level=fatal msg=”Error starting daemon: error initializing graphdriver: driver not supported”
Jan 22 21:30:07 ubuntu systemd[1]: docker.service: Main process exited, code=exited, status=1/FAILURE
Jan 22 21:30:07 ubuntu systemd[1]: docker.service: Unit entered failed state.
Jan 22 21:30:07 ubuntu systemd[1]: docker.service: Failed with result ‘exit-code’.

关于docker进程没起来的原因可能会有很多,比如：模块没加载，系统内核版本过低，硬盘空间不足等等。百度搜了一圈，国内大多都是抄来抄去，大多雷同，谷歌了一番，英文不太好，找了到一篇解决博客，参考：

http://log.rowanto.com/broken-docker-in-debian-jessie/

解决方法：

sudo mv /var/lib/docker/aufs /var/lib/docker/aufs.old
sudo service docker start

或者是执行：

docker -d -s aufs –storage-driver=overlay > /dev/null 2>&1 &

树莓派debian做简单的路由功能

写篇文章记录下，往后备查。主要参考：
https://wiki.debian.org/WiFi/
https://wiki.debian.org/HowToIdentifyADevice/USB
https://wiki.debian.org/rt2800usb
http://superuser.com/questions/597834/bridging-wifi-to-ethernet-on-ubuntu-not-working
http://blog.sina.com.cn/s/blog_6987dc7c01013tb5.html
http://bbs.csdn.net/topics/340028399/

先安装用到的工具：
apt-get install wireless-tools network-manager wpasupplicant firmware-ralink usbutils isc-dhcp-server
这里有说明下：
wireless-tools 无线网络扫描器及监控工具
network-manager 网络管理器
wpasupplicant WPA/WPA2和WAPI无线协议和加密认证
firmware-ralink 一些linux下的驱动固件
isc-dhcp-server dhcp服务软件

把连接的无线网共享给其他设备服务，先使其无线连接到路由，现在大多都是wpa2加密的。
sudo ip link set wlan0 up
wpa_passphrase TP-LINK_36EE ****** > ~/wpa_supplicant.conf
wpa_supplicant -i wlan0 -c ~/wpa_supplicant.conf
dhclient wlan0
现在没错就因该能上网了。

编辑/etc/network/interfaces,让它开机自动连接。加入代码：
auto wlan0
iface wlan0 inet dhcp
wpa-conf /root/wpa_supplicant.conf

配置一个简单的dhcp server，让它自动分配ip地址。
修改/etc/dhcp/dhcpd.conf,加入代码：
ddns-update-style none;
option domain-name “192.168.18.1”;
option domain-name-servers 221.228.255.1, 218.2.135.1;
default-lease-time 600;
max-lease-time 7200;
log-facility local7;
subnet 192.168.18.0 netmask 255.255.255.0 {
range 192.168.18.2 192.168.18.254;
option domain-name-servers 221.228.255.1, 218.2.135.1;
option domain-name “192.168.18.1”;
option routers 192.168.18.1;
option broadcast-address 192.168.18.255;
default-lease-time 600;
max-lease-time 7200;
}

再就是配置一下eth0接口的ip：
auto eth0
iface eth0 inet static
address 192.168.18.1
gateway 192.168.18.1
netmask 255.255.255.0
network 192.168.18.0
broadcast 192.168.18.255

开启dhcp服务：
systemctl enable isc-dhcp-server.service

开启路由转发功能：
echo net.ipv4.ip_forward=1 > /etc/sysctl.conf （永久有效）
echo 1> /proc/sys/ net/ipv4/ip_forward (临时有效）

最后把wlan0共享，使其eth0接口的设备能连接到互联网 ，把它加入到/etc/rc.local,让它开机自起。
iptables -t nat -A POSTROUTING -o wlan0 -j MASQUERADE

编译squid启用SSL

由于直接使用 apt 安装的 squid 没有启用 SSL，所以需要使用源代码编译安装。

首先需要安装 openssl 及相关的依赖。

apt-get install openssl
apt-get install libssl-dev
apt-get install ssl-cert

下载源码编译squid

apt-get source squid
apt-get build-dep squid
apt-get install devscripts build-essential fakeroot

squid 版本为 3.4.8

cd squid3-3.4.8

./configure 添加ssl的支持

./configure
–prefix=/usr/local/squid
–datadir=/usr/local/squid
–sysconfdir=/usr/local/squid/etc/
–mandir=/usr/local/squid/share/man
–enable-inline
–disable-arch-native
–enable-async-io=8
–enable-storeio=”ufs,aufs,diskd,rock”
–enable-removal-policies=”lru,heap”
–enable-delay-pools
–enable-cache-digests
–enable-icap-client
–enable-follow-x-forwarded-for
–enable-auth-basic=”DB,fake,getpwnam,LDAP,MSNT,MSNT-multi-domain,NCSA,NIS,PAM,POP3,RADIUS,SASL,SMB”
–enable-auth-digest=”file,LDAP”
–enable-auth-negotiate=”kerberos,wrapper”
–enable-auth-ntlm=”fake,smb_lm”
–enable-external-acl-helpers=”file_userip,kerberos_ldap_group,LDAP_group,session,SQL_session,unix_group,wbinfo_group”
–enable-url-rewrite-helpers=”fake”
–enable-eui
–enable-esi
–enable-icmp
–enable-zph-qos
–enable-ecap
–disable-translation
–with-swapdir=/usr/local/squid/var/spool/squid
–with-logdir=/usr/local/squid/var/log/squid
–with-pidfile=/usr/local/squid/var/run/squid.pid
–with-filedescriptors=65536
–with-large-files
–enable-ssl
–with-open-ssl=/etc/ssl/openssl.cnf
–with-openssl
–with-default-user=proxy

注意 with-open-ssl 的值为实际 openssl.cnf 所在路径

编译 squid （我这里懒得打包了，直接make && make install）

make -j4

安装

make install

主要参考文献:
https://wiki.ubuntu.com/ARM/BuildEABIChroot
http://blog.csdn.net/embbnux/article/details/12751465
http://blog.csdn.net/embbnux/article/details/17694101

首先是安装模拟器：

sudo apt-get update
sudo apt-get install qemu-user-static debootstrap

获取基本文件系统：
sudo qemu-debootstrap –arch armhf raring ubuntu_arm

等着下载完成；

然后进入该系统：

sudo chroot ubuntu_arm
uname -a

可以看到是arm7l

修改软件源：
vi /etc/apt/source.list

添加：

deb http://debian.ustc.edu.cn/ubuntu-ports/ trusty main
deb http://debian.ustc.edu.cn/ubuntu-ports/ ;trusty multiverse
deb http://debian.ustc.edu.cn/ubuntu-ports/ ;trusty restricted
deb http://debian.ustc.edu.cn/ubuntu-ports/ ;trusty universe

换成中科大的源，然后就可以安装大多软件了，just apt

现在可以安装些用到的软件了，我这安装个openssh-server。没有usb-ttl，没法串口，悲催！

这里有个警告：

解决方法参考了：

http://www.xuebuyuan.com/2041098.html

直接运行
echo “export LC_ALL=C” >> /root/.bashrc
然后执行一下：
source /root/.bashrc

配置一下openssh-server，让她可以root登录：
vi /etc/ssh/sshd_config
注释掉 #PermitRootLogin without-password
添加 PermitRootLogin yes

现在启用root超级用户：
sudo passwd root

下面配置一下网络：（因为没法串口登录配置，所以提前配置好）
我这里路由器有DHCP，就直接自动获取ip。
vi /etc/network/interfaces

auto eth0
iface eth0 inet dhcp

配置域名服务器：打开/etc/resolv.conf文件
echo “nameserver 127.0.1.1” >> /etc/resolv.conf

好了现在把文件系统拷到sd卡的ext4分区上，上真机让她跑起来吧！

启动成功：（300多兆的系统，比起官方的小好多了）
