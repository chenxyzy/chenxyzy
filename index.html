Windows 一些问题处理

Windows 图标异常

定位到以下路径：HKEY_CURRENT_USER\Software\Classes\Local Settings\Software\Microsoft\Windows\CurrentVersion\TrayNotify
删除PastIconsStream \解决通知栏图标显示异常
删除IconStreams
注意：重启Windows资源管理器生效

Windows 开启始终在任务栏上显示所有图标和通知

定位到HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer
修改EnableAutoTray DWORD值为0，表示开启任务栏显示所有图标和通知；1表示关闭
注意：重启Windows资源管理器生效

Windows 查看注册表对应的本地文件位置

定位到HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\hivelist

Windows7 注册表修改关闭防火墙

定位到HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\SharedAccess\Parameters\FirewallPolicy\PublicProfile
设置EnableFirewall的值为0
定位到HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\SharedAccess\Parameters\FirewallPolicy\StandardProfile
设置EnableFirewall的值为0
注意：重启生效

Windows 网卡驱动后面带#数字

解决驱动名后面带#数字：
定位到HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Network{4D36E972-E325-11CE-BFC1-08002BE10318}\Descriptions
删除Realtek PCIe GbE Family Controller
解决网络适配器连接信息里的本地连接后面带数字：
定位到HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\Control\Network{4D36E972-E325-11CE-BFC1-08002BE10318}
在下面的很多项里展开后查看Connection的Name，找到关键信息后重命名为想要的名称（实时生效），当然也可以删除其他不想要的本地连接信息

Windows10跳过创建用户直接用Administrator身份登录

1、部署完系统进入到区域设置后，按Shift + F10 开启命令行；
2、在命令行 输入 compmgmt，定位到“本地用户和组”并展开；选中用户定位到Administrator，将禁用属性去掉；
3、Ctrl + Shift + F3 进入审计模式（会自动重启）；
4、启动后关闭“系统准备工具XXX” 对话框；
5、将 C:\Windows\System32\svchost.exe 复制到 C:\Windows\System32\oobe\audit.exe（如果提示无权限，修改权限即可；原始audit.exe最好备份下）；
6、重启即可计入系统 shutdown -r -t 0；

    mkfs.ext4 /dev/sda9
    mkfs.xfs /dev/sda10
    mkswap /dev/sda11

    mount /dev/sda10 /mnt
    mkdir -p /mnt/boot/EFI
    mount /dev/sda9 /boot
    mount /dev/sda1 /mnt/boot/EFI
    swapon /dev/sda11

3，安装基本的软件包：
pacstrap /mnt base base-devel

4，生成fstab：
genfstab -U -p /mnt >> /mnt/etc/fstab

5，切换根目录：
arch-chroot /mnt /bin/bash

6，设置语言编码：
cat >> /etc/locale.gen << EOF
>en_US.UTF-8 UTF-8
>zh_CN.UTF-8 UTF-8
>EOF

locale-gen

echo LANG=en_US.UTF-8 >> /etc/locale.conf

7，设置时区：
mv /etc/localtime /etc/localtime.old
cp /usr/share/zoneinfo/Asia/Shanghai /etc/localtime

8，设置主机名，配置网络：
hostnamectl set-hostname ArchLinu
systemctl enable dhcpcd.service

9，生成initramfs：
mkinitcpio -p linux

10，安装grub：
pacman -S grub os-prober efibootngr
grub-install –efi-directory=/boot/EFI –bootloader-id=ArchLinu
grub-mkconfig -o /boot/grub/grub.cfg

11，安装基本软件包结束，退出当前根，卸载文件系统，并重启到archlinux：
exit
umount /mnt/boot/EFI
umount /mn/boot
umount /mnt
shutdown -r 0

12，安装xorg和显卡驱动：
pacman -S nvidia xf86-video-intel 我的笔记本是nvidia的显卡，而且较新，340xx和304xx试过驱动不了
这里用nvidia的配置工具生成的xorg.conf，启动X Windows直接死机;不配置就黑屏，解决办法是在家目录下的.xinitrc文件中加入：

    xrandr --setprovideroutputsource
    modesetting NVIDIA-0
    xrandr --auto   
   
    如果没有这个文件可以从/etc/X11/xinit/xinitrc复制到家目录的.xinitrc;   
   
pacman -S xorg-server xorg-xinit xorg-server-utils
注意：X Windows启动依赖xterm：
pacman -S xterm

cp /etc/xdg/autostart/fcitx-autostart.desktop ~/.config/autostart/

export GTK_IM_MODULE=fcitx

Firefox无声音：
Firefox 在linux上依赖pulseaudio;
pacman -S pulseaudio

参考： https://support.mozilla.org/zh-CN/kb/%E8%A7%86%E9%A2%91%E6%88%96%E9%9F%B3%E9%A2%91%E6%97%A0%E6%B3%95%E6%92%AD%E6%94%BE

Alsa系统声音相关：
pacman -S alsa-utils

配置默认声卡：
/etc/asound.conf or ~/.asoundrc

defaults.ctl.card 2
defaults.pcm.card 2
defaults.timer.card 2
后面的数字为声卡的索引

关闭独显：
echo 1 > /sys/bus/pci/devices/0000:01:00.0/remove
重新打开独显：
echo 1 > /sys/bus/pci/rescan
PS: use root user

调节显示屏亮度：
echo 10 > /sys/class/backlight/amdgpu_bl0/brightness
PS: amdgpu_bl0可能会根据显卡的不同变化

Postgresql编码报错

安装完毕odoo9 后出现报错:Database creation error: new encoding (UTF8) is incompatible with the encoding of the template database (SQL_ASCII) HINT: Use the same encoding as in the template database, or use template0 as template.

root@server:# su postgres
postgres@server: $ psql -U postgres
psql (9.0.3)
Type “help” for help.

postgres=# update pg_database set datallowconn = TRUE where datname = ‘template0’;
UPDATE 1
postgres=# \c template0
You are now connected to database “template0”.
template0=# update pg_database set datistemplate = FALSE where datname = ‘template1’;
UPDATE 1
template0=# drop database template1;
DROP DATABASE
template0=# create database template1 with template = template0 encoding = ‘UTF8’;
CREATE DATABASE
template0=# update pg_database set datistemplate = TRUE where datname = ‘template1’;
UPDATE 1
template0=# \c template1
You are now connected to database “template1”.
template1=# update pg_database set datallowconn = FALSE where datname = ‘template0’;
UPDATE 1
